The course

Our training explains everything you need to know in order to start using Concourse within a team.

The course is delivered on-site throughout one whole day, with a heavy focus on practical exercises to ensure that trainees leave the session able to use Concourse immediately. We provide multiple instructors to walk the floor during these practical exercises, giving trainees the opportunity to ask questions and leverage EngineerBetter's experience of running Concourse in production.

The course is programming-language agnostic - although some examples use Ruby to demonstrate testing a software product, no knowledge is required and the examples are simple and easy to understand.

What is Concourse?

Concourse is more than a regular continuous integration server.

Concourse features pipelines as a first-class citizen, allowing your organisation to visually map every resource and process that is involved in delivering value via software.

Concourse offers a predictable, repeatable and hassle-free CI experience.

How Does Concourse Work?

  • Declarative - all pipeline configuration is expressed in YAML files that can be kept under source control, allowing Pipelines-as-Code.
  • Immutable - configuration can’t be changed through the UI, preventing ‘snowflake’ CI servers that are hard to recreate in a disaster.
  • Isolated - all tasks run in containers, meaning that the output of one task cannot pollute or break another task or pipeline.
  • Externalised - tasks are expressed as scripts that live in your code repositories, making your project portable with no lock-in to Concourse.
  • Repeatable - all resources that provide input to a pipeline are versioned, allowing you to author idempotent and deterministic jobs.
  • Simple - there are no plug-ins, which makes operating Concourse simple and means a buggy collection of open source plug-ins won’t break your builds. Extensibility is instead achieved through optional container images specified by each pipeline.
